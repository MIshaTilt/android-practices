# Отчет по выполнению практических заданий №3

## Цель работы
Целью данной работы является изучение механизмов взаимодействия между компонентами Android-приложения с использованием объектов Intent, а также освоение работы с фрагментами и современными навигационными компонентами. В ходе выполнения заданий необходимо реализовать передачу данных между активностями через явные и неявные намерения, использовать Activity Result API для получения обратной связи от вызываемых компонентов и научиться работать с системными приложениями для выполнения стандартных действий, таких как совершение звонков или просмотр веб-страниц. Также важной частью работы является проектирование гибкого пользовательского интерфейса с использованием фрагментов, способного адаптироваться к изменению ориентации экрана, и создание комплексного проекта на основе шаблона Navigation Drawer с интеграцией WebView.

## Программная реализация
В первом модуле IntentApp реализован механизм передачи системного времени из основной активности во вторую. Для этого в MainActivity получается текущее время в миллисекундах, форматируется в строку с помощью SimpleDateFormat и помещается в объект Intent через метод putExtra. Во второй активности данные извлекаются методом getStringExtra и отображаются в TextView вместе с текстом, содержащим квадрат порядкового номера студента в списке группы. Этот пример демонстрирует базовое использование явных намерений для навигации и передачи простой информации между экранами приложения.

Модуль Sharer посвящен работе с неявными намерениями и использованию стандартного действия ACTION_SEND. В коде создается Intent, устанавливается MIME-тип данных и текст сообщения. Для того чтобы предоставить пользователю выбор из всех подходящих приложений на устройстве, используется метод Intent.createChooser. Это позволяет корректно обрабатывать ситуацию, когда несколько программ могут обработать запрос на отправку данных, и гарантирует появление диалогового окна выбора при каждом вызове.

В приложении FavoriteBook продемонстрировано использование современного Activity Result API для обмена данными между активностями. В MainActivity регистрируется обработчик результата через registerForActivityResult, который ожидает строку с названием книги. При нажатии на кнопку запускается ShareActivity, где пользователь вводит название своей любимой книги. После ввода данных используется метод setResult с кодом RESULT_OK для возврата информации обратно в родительскую активность. Такой подход является более надежным и гибким по сравнению с устаревшим методом startActivityForResult, так как он отделяет логику обработки результата от жизненного цикла самой активности.

Приложение SystemIntentsApp демонстрирует вызов системных функций через неявные намерения с использованием URI. Реализованы три метода для взаимодействия с внешними приложениями: совершение звонка через ACTION_DIAL с протоколом tel, открытие веб-страницы в браузере через ACTION_VIEW с протоколом http и отображение координат на карте через ACTION_VIEW с протоколом geo. Использование Uri.parse позволяет быстро сформировать необходимые данные для передачи операционной системе, которая автоматически подбирает наиболее подходящий обработчик для выполнения запроса.

В модуле SimpleFragmentApp реализовано динамическое управление фрагментами и поддержка различных ориентаций устройства. В вертикальном режиме MainActivity использует FragmentManager и FragmentTransaction для переключения между двумя фрагментами в контейнере FrameLayout при нажатии на кнопки. Для горизонтальной ориентации создан отдельный макет, где оба фрагмента размещены статично через тег fragment с использованием весов для разделения экранного пространства. Это обеспечивает адаптивность интерфейса и демонстрирует преимущества использования фрагментов для построения модульного дизайна.

Финальный проект MireaProject основан на шаблоне Navigation Drawer и включает использование Navigation Architecture Component. В рамках этого задания созданы два ключевых фрагмента: DataFragment и WebViewFragment. DataFragment содержит информацию об интересующей отрасли, оформленную в соответствии с принципами Material You с соблюдением отступов и цветовой схемы. WebViewFragment реализует простейший браузер, используя компонент WebView для загрузки и отображения веб-страницы по умолчанию. Настройка переходов между ними осуществляется через навигационный граф в XML-файле, а управление боковым меню реализовано с помощью NavController и AppBarConfiguration.

## Заключение
В результате выполнения практических работ были получены навыки работы с основными компонентами Android-приложений. Изучены различные способы использования Intent для реализации навигации и передачи данных, включая работу с системными намерениями и получение результатов от дочерних активностей. Отработаны техники создания гибких интерфейсов на базе фрагментов, что позволяет приложениям эффективно функционировать на устройствах с разными параметрами экрана. Использование шаблона Navigation Drawer и компонентов навигации позволило создать структурированное приложение с удобным пользовательским интерфейсом, объединяющее в себе различные функциональные модули.
