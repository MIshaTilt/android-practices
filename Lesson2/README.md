# Отчет по выполнению задания №2

## Цель работы
Основной целью работы является изучение механизмов взаимодействия между компонентами Android-приложения, исследование жизненного цикла Activity, а также освоение инструментов отладки и оповещения пользователя. Необходимо реализовать ряд модулей, демонстрирующих обработку событий жизненного цикла, вызов экранов с использованием явных и неявных намерений, а также использование различных типов диалоговых окон и уведомлений.

## Программная реализация
В первой части работы был создан модуль activitylifecycle, в котором были переопределены основные методы жизненного цикла класса MainActivity, такие как onCreate, onStart, onResume, onPause, onStop, onDestroy и onRestart. В каждом методе было добавлено логирование с использованием класса Log.i для отслеживания состояния активности в консоли Logcat при взаимодействии с приложением, например, при его сворачивании или повороте экрана. Это позволяет понять, как система управляет ресурсами и сохраняет состояние интерфейса.

В модуле multiactivity реализован механизм перехода между экранами с помощью явных намерений. На главный экран была добавлена кнопка, нажатие которой инициирует создание объекта Intent с указанием класса SecondActivity. Передача данных между активностями была продемонстрирована через метод putExtra, что позволило передать текстовую строку на второй экран. В SecondActivity полученные данные извлекаются через метод getIntent и отображаются пользователю.

Модуль intentfilter демонстрирует работу с неявными намерениями, где целевая активность не указывается напрямую, а описывается действие и тип данных. Например, для открытия веб-сайта использовался Intent.ACTION_VIEW с указанием URI-адреса, а для отправки текстовых данных в другие приложения — Intent.ACTION_SEND с возможностью выбора целевого приложения через Intent.createChooser.

В модулях toastapp, notificationapp и dialog реализованы различные способы взаимодействия с пользователем. Класс Toast был использован для вывода коротких всплывающих сообщений с информацией о количестве символов в текстовом поле. Для более сложных уведомлений в системной панели использовались NotificationManager и NotificationChannel, обеспечивающие информирование пользователя даже при свернутом приложении. Для модального взаимодействия применен класс DialogFragment и AlertDialog.Builder, позволяющие создавать диалоговые окна с кнопками выбора. Дополнительно были реализованы специализированные диалоги TimePickerDialog, DatePickerDialog и кастомный диалог прогресса для решения более специфичных задач ввода.

## Заключение
В ходе выполнения практической работы были освоены фундаментальные аспекты разработки Android-приложений. Получены знания о жизненном цикле активности, что критически важно для создания стабильных и эффективных с точки зрения использования ресурсов приложений. Приобретены практические навыки организации навигации между экранами и интеграции с другими системными приложениями через намерения. Успешная реализация различных типов уведомлений и диалогов позволила создать более информативный и интерактивный пользовательский интерфейс.
