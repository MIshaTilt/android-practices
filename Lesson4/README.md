# Отчет по практической работе №4
**Тема:** Разработка мобильных приложений. Асинхронная работа и фоновые процессы в ОС Android.

### 1. Цель работы
Изучить принципы многопоточности и концепции асинхронной работы в операционной системе Android. Освоить различные механизмы выполнения ресурсоемких задач вне главного (UI) потока во избежание блокировки интерфейса (ошибки ANR). На практике применить механизмы `Thread`, `Looper` и `Handler`, устаревший API `Loader`, фоновые службы `Service` (Foreground Service) и современную библиотеку от Google — `WorkManager`.

---

### 2. Программная реализация

В ходе работы был создан проект, включающий несколько независимых модулей, каждый из которых демонстрирует определенный подход к работе с фоновыми потоками. Во всех модулях для связи XML-разметки с Java-кодом использовался современный инструмент **ViewBinding**.

#### 2.1. Модуль `thread` (Базовая работа с потоками)
**Описание:** Реализовано приложение, вычисляющее среднее количество пар в день за месяц. Вычисления производятся в отдельном потоке.
**Объяснение кода:**
Был создан экземпляр класса `Thread`, внутри которого переопределен метод `run()`. Главная особенность платформы Android заключается в том, что обновлять графический интерфейс можно **только из главного (Main/UI) потока**. Для безопасной передачи результата вычислений на экран был использован метод `runOnUiThread()`.
```java
new Thread(new Runnable() {
    @Override
    public void run() {
        // ... фоновые вычисления ...
        float result = pairs / days;
        runOnUiThread(new Runnable() {
            @Override
            public void run() {
                binding.textResult.setText("Результат: " + result);
            }
        });
    }
}).start();
```

#### 2.2. Модуль `data_thread` (Способы взаимодействия с UI)
**Описание:** Изучены альтернативные методы передачи Runnable-задач в очередь главного потока.
**Объяснение кода:**
Для элемента `TextView` были поочередно вызваны методы `.post()` (добавление задачи в конец очереди UI-потока) и `.postDelayed()` (выполнение задачи с временной задержкой). Это позволяет планировать обновление интерфейса из фона без явного вызова `runOnUiThread`.

#### 2.3. Модуль `looper` (Очередь сообщений)
**Описание:** Реализована двусторонняя связь между потоками. Пользователь вводит возраст и профессию, эти данные отправляются в фоновый поток (`MyLooper`), который "засыпает" на количество секунд, равное возрасту, после чего возвращает ответ.
**Объяснение кода:**
Класс `MyLooper` использует `Looper.prepare()` и `Looper.loop()`, что превращает обычный поток в бесконечный цикл обработки сообщений. Для отправки и приема сообщений использовался класс `Handler`. Данные упаковывались в объект `Message` с помощью `Bundle`.
```java
// Отправка данных из MainActivity в фоновый Looper
Message msg = Message.obtain();
Bundle bundle = new Bundle();
bundle.putString("WORK", workStr);
bundle.putInt("AGE", age);
msg.setData(bundle);
myLooper.mHandler.sendMessage(msg);
```

#### 2.4. Модуль `CryptoLoader` (Загрузчики)
**Описание:** Реализовано шифрование текста алгоритмом AES. Зашифрованный текст и ключ передаются в компонент `Loader`, который в фоне расшифровывает данные и возвращает результат в `Activity`.
**Объяснение кода:**
Создан класс-наследник `AsyncTaskLoader`. Для его запуска используется `LoaderManager`. Данный компонент автоматически учитывает жизненный цикл активности, предотвращая утечки памяти. Вся тяжелая работа (дешифровка) выполняется в переопределенном методе `loadInBackground()`.
```java
// Выполнение дешифровки в фоновом потоке Loader-а
@Override
public String loadInBackground() {
    SystemClock.sleep(2000); // Имитация долгой работы
    SecretKey originalKey = new SecretKeySpec(key, 0, key.length, "AES");
    return decryptMsg(cryptText, originalKey);
}
```

#### 2.5. Модуль `ServiceApp` (Фоновые сервисы - Музыкальный плеер)
**Описание:** Разработан музыкальный проигрыватель на основе компонента `Service`. Музыка продолжает играть даже при сворачивании приложения.
**Объяснение кода:**
Был реализован Foreground Service (сервис переднего плана). Для этого в `onCreate()` сервиса создается неудаляемое уведомление (Notification), привязанное к сервису. Были учтены новые политики безопасности Android 14+, требующие явного указания типа сервиса (`FOREGROUND_SERVICE_TYPE_MEDIA_PLAYBACK`) и запроса разрешения `POST_NOTIFICATIONS` у пользователя. За воспроизведение отвечал класс `MediaPlayer`.

#### 2.6. Модуль `work_manager` (Гарантированное выполнение задач)
**Описание:** Изучена библиотека WorkManager. Создана задача, требующая особых системных условий для своего запуска.
**Объяснение кода:**
Создан класс `UploadWorker`, наследуемый от `Worker`, реализующий метод `doWork()`. В `MainActivity` перед постановкой задачи в очередь были заданы строгие ограничения (Constraints): наличие подключения к Wi-Fi (`UNMETERED`) и нахождение устройства на зарядке (`setRequiresCharging(true)`).
```java
Constraints constraints = new Constraints.Builder()
        .setRequiredNetworkType(NetworkType.UNMETERED)
        .setRequiresCharging(true)
        .build();

OneTimeWorkRequest request = new OneTimeWorkRequest.Builder(UploadWorker.class)
        .setConstraints(constraints)
        .build();
WorkManager.getInstance(this).enqueue(request);
```

### 3. Заключение

В ходе выполнения практической работы были успешно изучены и применены на практике основные способы реализации многопоточности в ОС Android. По результатам работы можно сделать следующие выводы об области применения изученных инструментов:

1. **Thread / Runnable** — подходят для простейших разовых задач, однако управлять ими вручную сложно, особенно при повороте экрана (смене конфигурации).
2. **Looper и Handler** — мощный инструмент для создания очередей сообщений и фоновых потоков, которые должны постоянно "слушать" новые команды.
3. **Loader** — эффективный (хоть и устаревший) инструмент для асинхронной загрузки данных, неразрывно связанный с жизненным циклом компонентов (в современной разработке заменен на ViewModel + Coroutines/RxJava).
4. **Service (Foreground)** — идеален для длительных задач, о которых пользователь должен знать (например, воспроизведение музыки, навигатор, таймер).
5. **WorkManager** — наиболее современный, гибкий и надежный стандарт для фоновых задач (синхронизация, выгрузка логов, бэкапы). Он гарантирует выполнение кода даже в случае перезагрузки устройства и позволяет гибко настраивать системные ограничения для сохранения заряда батареи.

Задание выполнено в полном объеме, ошибки ANR (Application Not Responding) в процессе ресурсоемких операций исключены.
