# Отчет по практической работе №5
**Тема:** Основы использования аппаратных возможностей мобильных устройств

### 1. Цель работы
Целью данной работы является изучение программных способов взаимодействия с аппаратным обеспечением современных мобильных устройств на базе ОС Android. В ходе работы необходимо освоить:
*   Получение списка доступных датчиков (сенсоров) устройства и их характеристик.
*   Реализацию взаимодействия с датчиком акселерометра для определения положения устройства в пространстве.
*   Применение системы разрешений (`Runtime Permissions`) для доступа к конфиденциальным ресурсам (камера, микрофон).
*   Использование API `MediaStore` и `FileProvider` для вызова системного приложения «Камера» и сохранения медиафайлов.
*   Работу с мультимедийными компонентами `MediaRecorder` и `MediaPlayer` для записи и воспроизведения звука.

### 2. Программная реализация

#### 2.1. Список датчиков
Для получения доступа к датчикам был использован класс `SensorManager`. С помощью метода `getSensorList(Sensor.TYPE_ALL)` был сформирован список всех поддерживаемых датчиков. Полученные данные (имя и максимальный диапазон датчика) были выведены в `ListView` посредством `SimpleAdapter`.

#### 2.2. Акселерометр
Для отслеживания изменений положения устройства был реализован интерфейс `SensorEventListener`. Основная логика вынесена в метод `onSensorChanged`, который фиксирует ускорение по осям X, Y и Z. Регистрация слушателя происходит в `onResume`, а отмена — в `onPause`, что позволяет экономить ресурсы аккумулятора.

#### 2.3. Механизм разрешений
Для корректной работы с камерой и микрофоном был внедрен механизм проверки разрешений `ContextCompat.checkSelfPermission`. В случае отсутствия доступа вызывается метод `ActivityCompat.requestPermissions`. Обработка ответа пользователя реализована в `onRequestPermissionsResult`.

#### 2.4. Камера
Для реализации функционала камеры был использован `Intent` с действием `MediaStore.ACTION_IMAGE_CAPTURE`. Для обеспечения безопасности передачи файлов между приложениями настроен `FileProvider` (в манифесте и XML-файле `paths.xml`), который генерирует защищенный URI для сохранения снимка во внутреннем хранилище приложения. Использование `ViewBinding` позволило эффективно связать макет `activity_main.xml` с кодом логики.

#### 2.5. Микрофон и MediaRecorder
Реализация диктофона базируется на конечном автомате `MediaRecorder`. Настройка параметров (источник `MIC`, формат `THREE_GPP`, кодек `AMR_NB`) позволила создать работоспособное приложение для записи аудио. Воспроизведение реализовано через класс `MediaPlayer`. Переключение состояний кнопок записи и воспроизведения предотвращает конфликты доступа к ресурсу микрофона.

### 3. Заключение
В ходе выполнения практической работы я изучил принципы работы с аппаратными компонентами мобильного устройства. 
*   **Освоены:** методы работы с сенсорами, работа с жизненным циклом компонентов записи аудио и видео.
*   **Изучены:** механизмы обеспечения приватности данных через систему разрешений и безопасная передача файлов через `FileProvider`.
*   **Результат:** созданы четыре программных модуля, каждый из которых решает специфическую задачу взаимодействия с железом смартфона. Полученные знания позволяют разрабатывать приложения, использующие датчики и мультимедиа-ресурсы устройства согласно актуальным требованиям безопасности ОС Android.
