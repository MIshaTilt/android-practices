# Отчет по выполнению задания

## Цель работы
Целью данной работы является изучение основ разработки мобильных приложений для операционной системы Android с использованием среды проектирования Android Studio. В ходе выполнения заданий необходимо было освоить принципы настройки рабочего окружения, создания проектов и модулей, а также изучить внутреннюю структуру проекта и систему автоматизированной сборки Gradle. Особое внимание уделялось работе с графическим интерфейсом пользователя, изучению различных типов контейнеров Layout, механизмам адаптации интерфейса при изменении ориентации экрана и программному взаимодействию с элементами управления через обработчики событий.

## Программная реализация
На начальном этапе была произведена установка и конфигурация среды Android Studio, включая загрузку необходимых компонентов SDK и создание эмулятора в менеджере виртуальных устройств для тестирования приложений. Был создан базовый проект и ряд модулей, таких как layouttype, control_lesson1 и buttonclicker, в которых велась основная разработка. В файлах разметки XML были реализованы различные варианты пользовательского интерфейса с применением LinearLayout для линейного расположения элементов, TableLayout для формирования табличных структур и ConstraintLayout для создания гибких интерфейсов на основе системы привязок компонентов друг к другу и границам экрана. Каждому модулю соответствует свой файл build.gradle, где были настроены идентификаторы приложений, версии SDK и зависимости библиотек.

В процессе разработки была реализована поддержка различных ориентаций устройства. Для этого в ресурсах проекта создавались альтернативные файлы разметки в специальных директориях, что позволило адаптировать интерфейс под альбомный режим и обеспечить видимость всех элементов при повороте экрана. Программная логика была сосредоточена в классах Activity, где метод setContentView связывал Java-код с соответствующим XML-файлом разметки. С помощью метода findViewById осуществлялся поиск элементов интерфейса по их уникальным идентификаторам, что позволило программно изменять текст в полях TextView, надписи на кнопках и состояние флажков CheckBox.

Особое внимание в реализации было уделено обработке пользовательских нажатий в модуле buttonclicker. Был применен подход с использованием анонимных классов для реализации интерфейса View.OnClickListener, который регистрировался для кнопок методом setOnClickListener. Также был изучен альтернативный способ декларативного описания обработчика через атрибут onClick в XML-файле с последующим определением логики в методе Activity. Эти механизмы позволили реализовать интерактивность приложения, при которой нажатие на кнопку приводит к динамическому обновлению текстовой информации на экране и изменению состояния других графических компонентов.

## Заключение
В ходе выполнения работы были успешно освоены базовые инструменты и методологии разработки приложений под платформу Android. Была изучена структура проекта, назначение манифеста и ресурсных файлов, а также получены практические навыки верстки интерфейсов различной сложности. Реализация программного взаимодействия с элементами экрана и обработка событий позволили закрепить понимание принципов работы жизненного цикла Activity и механизмов обратной связи в мобильных интерфейсах. Полученные знания являются фундаментом для дальнейшего изучения архитектуры Android и разработки более сложных функциональных модулей.
