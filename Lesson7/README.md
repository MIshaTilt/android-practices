


# Отчет по практической работе № 7

**Тема:** Сетевое взаимодействие в OS Android и сторонние библиотеки

### Цель работы
Изучить и на практике применить различные подходы к организации сетевого взаимодействия в операционной системе Android.

В ходе выполнения работы необходимо было реализовать:
1. Низкоуровневое сетевое взаимодействие с использованием сокетов (`Socket`) для получения точного времени.
2. Взаимодействие по протоколу HTTP с использованием встроенного класса `HttpURLConnection` для отправки GET-запросов, получения и парсинга JSON-данных (определение местоположения по IP и запрос погоды по координатам).
3. Интеграцию сторонней облачной платформы Firebase (`FirebaseAuth`) для реализации полноценной системы регистрации, авторизации и верификации пользователей по электронной почте и паролю.

### Программная реализация

Для выполнения задания был создан Android-проект, включающий три отдельных модуля. Во всех модулях в манифесте было добавлено разрешение на использование интернета (`<uses-permission android:name="android.permission.INTERNET" />`), а для доступа к UI-элементам применялся механизм `ViewBinding`. Так как сетевые запросы нельзя выполнять в главном (UI) потоке, применялся механизм асинхронных задач.

#### 1. Модуль `TimeService` (Работа с сокетами)
В данном модуле реализовано подключение к серверу времени `time.nist.gov` по порту `13`.
*   **Реализация:** Был создан вспомогательный класс `SocketUtils`, содержащий метод `getReader`, который оборачивает `InputStream` сокета в `BufferedReader` для удобного чтения строковых данных.
*   **Объяснение кода:** В `MainActivity` был создан класс `GetTimeTask`, наследующийся от `AsyncTask`. В методе `doInBackground` создается экземпляр `Socket(host, port)`. Программа читает входящий поток. Так как сервер возвращает неформатированную строку вида `60067 23-05-03 19:50:23...`, в методе `onPostExecute` был добавлен алгоритм парсинга: строка разбивается по пробелам с помощью метода `split(" ")`. Из полученного массива извлекаются элементы с индексами 1 и 2, что позволяет вывести на экран дату и время в удобочитаемом виде. После завершения чтения сокет закрывается методом `close()`.

#### 2. Модуль `HttpURLConnection` (Работа с REST API и JSON)
Модуль выполняет каскадный сетевой запрос: сначала узнает координаты устройства, а затем по ним запрашивает погоду. Дополнительно в манифест добавлено разрешение `ACCESS_NETWORK_STATE` для проверки наличия подключения к сети.
*   **Реализация:** Перед выполнением запроса с помощью `ConnectivityManager` проверяется статус интернет-соединения. Если сеть доступна, запускается первая задача `DownloadIpTask`.
*   **Код:**
    *   Создан универсальный метод `downloadInfo`, который принимает URL, открывает `HttpURLConnection`, настраивает таймауты и метод запроса (`GET`). Входящий байтовый поток считывается через `ByteArrayOutputStream` и преобразуется в строку.
    *   В `DownloadIpTask` происходит запрос к `https://ipinfo.io/json`. В методе `onPostExecute` полученная строка преобразуется в объект `JSONObject`. С помощью методов `getString("city")` и `getString("loc")` извлекаются город и координаты (приходят склеенными через запятую, например "52.52,13.41").
    *   Координаты разделяются с помощью `split(",")` и подставляются в новый URL сервиса погоды Open-Meteo.
    *   Запускается вторая задача `DownloadWeatherTask`, которая обращается к API погоды, снова парсит JSON (извлекает вложенный объект `current_weather` и его поле `temperature`) и выводит результат в `TextView`.

#### 3. Модуль `FirebaseAuth` (Аутентификация Firebase)
В этом модуле реализован графический интерфейс и логика для управления учетными записями пользователей.
*   **Реализация:** Проект был привязан к Firebase через встроенный ассистент Android Studio, в `build.gradle` добавлены зависимости Firebase SDK. В консоли Firebase активирован провайдер входа "Электронная почта/пароль".
*   **Код:**
    *   В `MainActivity` инициализируется объект `FirebaseAuth.getInstance()`.
    *   **Управление UI:** Создан метод `updateUI(FirebaseUser user)`. Если пользователь авторизован (`user != null`), метод скрывает поля ввода и кнопки регистрации (`setVisibility(View.GONE)`), и показывает кнопки выхода и верификации, а также выводит уникальный `UID` пользователя. Если не авторизован — возвращает экран к первоначальному состоянию ввода данных.
    *   **Валидация:** Реализован метод `validateForm()`, проверяющий, не пусты ли поля `email` и `password`.
    *   **Регистрация и Вход:** Используются асинхронные методы `mAuth.createUserWithEmailAndPassword()` и `mAuth.signInWithEmailAndPassword()`. На них вешаются слушатели `addOnCompleteListener`. В случае успеха вызывается `updateUI(user)`, в случае ошибки — выводится `Toast` с предупреждением.
    *   **Верификация:** По нажатию на кнопку вызывается метод `user.sendEmailVerification()`, который отправляет реальное письмо с подтверждением на указанный email.
    *   **Сохранение сессии:** В жизненном цикле активности (в методе `onStart()`) вызывается `mAuth.getCurrentUser()`. Это позволяет приложению "помнить" авторизованного пользователя даже после перезапуска приложения.
